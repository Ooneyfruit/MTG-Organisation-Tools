# UI Style Guidelines

This workspace requires that all user interfaces look like standard, native Tkinter widgets.

## Tkinter UI Styling Rules

1. **Standard Look and Feel**:
   - Avoid implementing complex themes, custom flat styles, heavy modern color configurations, or third-party style packages (like ttkthemes or custom modern themes) unless explicitly requested.
   - Use standard Tkinter and standard TTK elements with their default appearance.
2. **Minimal Styling**:
   - Do not override colors with custom hex codes unless necessary for clarity (e.g., error red or success green).
   - Let the operating system render the default styling/theme of the widgets (standard borders, buttons, and colors).
3. **Layout**:
   - Keep layouts simple, clean, and functional.
